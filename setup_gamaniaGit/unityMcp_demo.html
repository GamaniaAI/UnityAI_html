1. 場景管理（Scene Management）

此區展示整個 Unity 場景的基本狀態與管理能力。 觀者可看到目前場景的載入狀態、儲存結果，確認場景是否正確建立。

成功指標：場景資訊文字正確顯示。

2. GameObject 操作

此區展示 GameObject 的建立、修改、刪除與搜尋能力。 透過規則排列的立方體，可以直觀觀察物件是否成功被建立與操作。

成功指標：物件出現 / 消失、位置變化、搜尋結果高亮顯示。

3. 組件管理（Component Management）

此區展示對 GameObject 組件的動態管理能力，例如新增或移除 Rigidbody。 組件存在與否會直接影響物件行為。

成功指標：物件出現下落、停止或行為改變。

4. 資源與材質管理

此區展示 Unity 資源的搜尋、載入與材質套用能力。 透過材質顏色與外觀變化，觀者可以直接確認資源是否正確應用。

成功指標：物件外觀在套用材質前後明顯不同。

5. 腳本操作

此區展示腳本的建立、修改、驗證與刪除流程。 腳本是否成功編譯，會直接反映在物件的動態行為上。

成功指標：物件開始旋轉或運動，刪除腳本後行為停止。

6. VFX 與特效系統

此區展示視覺特效（粒子系統）的建立與參數調整能力。 粒子效果可以即時開關與修改。

成功指標：粒子出現、消失或密度與形態明顯改變。

7. 編輯器狀態控制

此區展示 Unity 編輯器模式（Edit / Play）的切換控制。 中央狀態文字會顯示目前所處的模式。

成功指標：狀態文字正確顯示目前模式，動態行為同步變化。

8. 批量操作

此區展示大量物件同時建立與操作的能力。 可觀察批量操作與單一操作在速度與結果上的差異。

成功指標：大量物件瞬間產生，排列規則一致。

9. 測試執行（Testing）

此區展示測試流程的啟動與結果回饋。 測試狀態與通過 / 失敗結果會以文字形式呈現。

成功指標：測試進度與結果清楚顯示。

10. 監控與控制台

此區展示 Unity 控制台訊息的即時讀取能力。 錯誤與警告會以文字形式顯示，協助快速定位問題。

成功指標：錯誤或訊息即時顯示並與操作對應。

11. Prefab 管理

此區展示將場景中的 GameObject 轉換為 Prefab 資產並保存的流程。 用來驗證 MCP 是否能正確操作 Unity 的資產化流程。

成功指標：Prefab 資產成功出現在指定資料夾中。

12. ScriptableObject 管理

此區展示 ScriptableObject 的建立與參數設定能力。 用來驗證 MCP 對純資料資產的操作能力。

成功指標：ScriptableObject 成功建立且參數值正確。

13. 系統重新整理與狀態讀取

此區展示 Unity 資源重新整理後，編輯器狀態的即時回饋。 用來確認 MCP 與 Unity Editor 狀態同步是否正常。

成功指標：重新整理完成，狀態文字正確顯示。
